== Introduction

teavm-react is a library that lets you write https://react.dev[React 18] applications in *Java* and *Kotlin*, compiled to JavaScript via https://teavm.org[TeaVM].

You get full React capabilities -- components, hooks, context, memo, refs -- with complete type safety, IDE support, and zero JavaScript knowledge required.

=== Modules

[cols="1,3"]
|===
|Module |Purpose

|`teavm-react-core`
|Java API bindings to React 18: hooks, HTML DSL, builder DSL, class-based components, events, context.

|`teavm-react-kotlin`
|Idiomatic Kotlin wrapper: delegated state properties, lambda-with-receiver HTML builder, `fc()` function components, coroutine integration.

|`teavm-react-demo`
|Kitchen-sink demo application exercising every library feature.
|===

== Prerequisites

* *JDK 21* (build-time; source/target compatibility is Java 11)
* *Maven 3.8+*
* A *modern web browser* (React 18 is loaded via CDN at runtime)
* For Kotlin development: no extra tooling needed (the `kotlin-maven-plugin` is configured in the project)


== Quick Start

=== 1. Create the Java entry point

[source,java]
----
package com.example;

import ca.weblite.teavmreact.core.React;
import ca.weblite.teavmreact.core.ReactDOM;
import ca.weblite.teavmreact.core.ReactElement;
import ca.weblite.teavmreact.hooks.Hooks;
import ca.weblite.teavmreact.hooks.StateHandle;
import org.teavm.jso.JSObject;
import org.teavm.jso.dom.html.HTMLDocument;

import static ca.weblite.teavmreact.html.Html.*;

public class App {
    public static void main(String[] args) {
        var root = ReactDOM.createRoot(
            HTMLDocument.current().getElementById("root"));
        root.render(component(App::render, "App"));
    }

    static ReactElement render(JSObject props) {
        StateHandle<Integer> count = Hooks.useState(0);
        return div(
            h1("Hello teavm-react!"),
            p("Count: " + count.getInt()),
            button("Increment")
                .onClick(e -> count.updateInt(n -> n + 1))
                .build()
        );
    }
}
----

=== The TeaVM JS Interop Layer

teavm-react bridges Java and JavaScript using three TeaVM annotations:

==== `@JSBody` -- Inline JavaScript

Embeds raw JavaScript inside a Java method. This is how all React API calls are made.

[source,java]
----
@JSBody(params = {"type", "props"}, script =
    "return React.createElement(type, props);")
public static native ReactElement createElement(String type, JSObject props);
----

==== `@JSFunctor` -- Java Lambdas as JS Functions

Marks a functional interface so TeaVM passes its implementation as a raw JavaScript function (not a Java wrapper object).

[source,java]
----
@JSFunctor
public interface EventHandler extends JSObject {
    void handleEvent(SyntheticEvent event);
}
----

==== `@JSProperty` -- Interface Property Access

Maps Java interface getter methods to JavaScript property access.

[source,java]
----
public interface ChangeEvent extends SyntheticEvent {
    // Inherits getTarget() from SyntheticEvent → returns EventTarget
}
----

=== Critical Implementation Details

[IMPORTANT]
====
*No ES6 spread in `@JSBody`*: TeaVM's `@JSBody` does not support the `...` spread operator. The library uses `.apply/.concat` as a workaround:

[source,java]
----
// Instead of: React.createElement(type, props, ...children)
@JSBody(params = {"type", "props", "children"}, script =
    "return React.createElement.apply(null, " +
    "[type, props].concat(Array.prototype.slice.call(children)));")
----
====

[IMPORTANT]
====
*Event handlers must use dedicated setter methods*: Passing a `@JSFunctor` via a generic `setProperty` would wrap it in a Java object instead of passing the raw JS function. Always use the dedicated methods like `React.setOnClick()`:

[source,java]
----
// CORRECT — raw JS function is passed
React.setOnClick(props, handler);

// WRONG — would wrap the function in a Java object
React.setProperty(props, "onClick", handler);  // DO NOT DO THIS
----
====


== Java API: Functional Components (Approach A)

The functional approach mirrors idiomatic React.js. You define components as static methods, use `Html.*` static imports for elements, and call `Hooks.*` for state and effects.

=== Defining a Component

A component is any method matching the `RenderFunction` signature: it takes a `JSObject props` parameter and returns a `ReactElement`. Register it with `React.wrapComponent()`:

[source,java]
----
import static ca.weblite.teavmreact.html.Html.*;

JSObject myComponent = React.wrapComponent(App::myRender, "MyComponent");

static ReactElement myRender(JSObject props) {
    return div(
        h1("Hello World"),
        p("This is a functional component.")
    );
}
----

=== Using State

[source,java]
----
static ReactElement counter(JSObject props) {
    StateHandle<Integer> count = Hooks.useState(0);

    return div(
        h2("Count: " + count.getInt()),
        button("Increment")
            .onClick(e -> count.updateInt(n -> n + 1))
            .build(),
        button("Reset")
            .onClick(e -> count.setInt(0))
            .build()
    );
}
----

`StateHandle<T>` provides typed getters and setters:

* `getInt()`, `setInt(int)`, `updateInt(IntUpdater)`
* `getString()`, `setString(String)`, `updateString(StringUpdater)`
* `getBool()`, `setBool(boolean)`
* `getDouble()`, `setDouble(double)`
* `getDouble()`, `setDouble(double)`

=== Using Effects

[source,java]
----
static ReactElement timer(JSObject props) {
    StateHandle<Integer> seconds = Hooks.useState(0);

    // Run once on mount (empty deps), clean up on unmount
    Hooks.useEffect(() -> {
        var id = setInterval(() -> seconds.updateInt(n -> n + 1), 1000);
        return () -> clearInterval(id);  // cleanup
    }, Hooks.deps());  // empty array = mount only

    return p("Elapsed: " + seconds.getInt() + "s");
}
----

* `Hooks.useEffect(callback)` -- runs after every render
* `Hooks.useEffect(callback, Hooks.deps())` -- runs once on mount
* `Hooks.useEffect(callback, Hooks.deps(dep1, dep2))` -- runs when deps change

The callback returns a `VoidCallback` for cleanup, or `null` for no cleanup.

=== Interactive Elements with ElementBuilder

`Html.button()`, `Html.input()`, `Html.a()`, `Html.textarea()`, `Html.select()`, and `Html.img()` return an `ElementBuilder` for fluent prop chaining. Call `.build()` to finalize:

[source,java]
----
// Button with click handler
button("Click me")
    .onClick(e -> doSomething())
    .className("btn-primary")
    .disabled(false)
    .build()

// Text input with controlled value
input("text")
    .value(name.getString())
    .onChange(e -> name.setString(e.getTarget().getValue()))
    .placeholder("Enter name")
    .build()

// Link
a("Visit site")
    .href("https://example.com")
    .target("_blank")
    .build()
----

=== Rendering Sub-Components

[source,java]
----
// Register a component once (typically as a static field)
static final JSObject counter = React.wrapComponent(App::counterRender, "Counter");

// Render it as a child element
static ReactElement app(JSObject props) {
    return div(
        h1("My App"),
        Html.component(counter),            // no props
        Html.component(counter, myProps)     // with props
    );
}
----

[source,java]

List<String> items = List.of("Apple", "Banana", "Cherry");

return ul(
    Html.mapToElements(items, item -> li(item))

== Java API: Builder DSL (Approach B)

The builder DSL provides a Java-idiomatic fluent pattern using `DomBuilder` inner classes. Each HTML element type has a dedicated class with a `create()` factory method.

=== Basic Usage

[source,java]
----
import ca.weblite.teavmreact.html.DomBuilder.*;

ReactElement view = Div.create()
    .className("container")
    .child(H1.create().text("Hello World").build())
    .child(P.create().text("Built with the builder DSL.").build())
    .child(Button.create()
        .text("Click me")
        .onClick(e -> count.updateInt(n -> n + 1))
        .build())
    .build();
----

*Layout:* `Div`, `Span`, `Section`, `Article`, `Aside`, `Header`, `Footer`, `Main`, `Nav`

*Headings:* `H1`, `H2`, `H3`, `H4`, `H5`, `H6`

*Text:* `P`, `Pre`, `Code`, `Blockquote`, `Em`, `Strong`, `Small`

*Lists:* `Ul`, `Ol`, `Li`, `Dl`, `Dt`, `Dd`

*Table:* `Table`, `Thead`, `Tbody`, `Tfoot`, `Tr`, `Th`, `Td`, `Caption`

*Form:* `Form`, `Fieldset`, `Legend`, `Label`, `Button`, `Input`, `Select`, `Textarea`

*Media:* `A`, `Img`, `Figure`, `Figcaption`

*Misc:* `Hr`, `Br`, `Details`, `Summary`

=== Input Convenience Methods

The `Input` class provides typed factory methods:

[source,java]
----
Input.text()        // <input type="text">
Input.password()    // <input type="password">
Input.checkbox()    // <input type="checkbox">
Input.number()      // <input type="number">
----

=== Chaining Methods

All builders inherit from `DomBuilder`:

* `.text(String)` -- set text content
* `.child(ReactElement)` / `.child(DomBuilder)` / `.child(ElementBuilder)` -- add children
* `.className(String)` / `.id(String)` / `.key(String|int)`
* `.style(Style)` -- inline styles (see `Style.create()`)
* `.onClick(EventHandler)` / `.onChange(ChangeEventHandler)` / `.onKeyDown(KeyboardEventHandler)`
* `.onFocus(FocusEventHandler)` / `.onBlur(FocusEventHandler)` / `.onSubmit(SubmitEventHandler)`
* `.value(String)` / `.placeholder(String)` / `.disabled(boolean)` / `.type(String)`
* `.href(String)` / `.src(String)` / `.alt(String)`
* `.prop(String, String|int|boolean|double)` -- arbitrary property
* `.forEach(List, mapper)` / `.forEachElement(List, mapper)` -- list iteration
* `.build()` -- finalize the element

== Java API: Class-Based Components (Approach C)

The class-based approach is familiar to Swing/JavaFX developers. Extend `ReactView`, override `render()`, and declare state as field initializers.

=== Basic Counter

[source,java]
----
public class CounterView extends ReactView {
    private final StateHandle<Integer> count = Hooks.useState(0);

    @Override
    protected ReactElement render() {
        return Html.div(
            Html.h2("Count: " + count.getInt()),
            Html.button("Increment")
                .onClick(e -> count.updateInt(n -> n + 1))
                .build()
        );
    }
}
----

=== Rendering a ReactView

[source,java]
----
// As a ReactElement (most common)
ReactView.view(CounterView::new, "CounterView")
----

=== Lifecycle Methods

Override `onMount()` and `onUnmount()` for setup and teardown:

[source,java]
----
public class StopwatchView extends ReactView {
    private final StateHandle<Integer> elapsed = Hooks.useState(0);
    private final StateHandle<Boolean> running = Hooks.useState(false);

    @Override
    protected void onMount() {
        // called once when the component mounts
    }

    @Override
    protected void onUnmount() {
        // called when the component unmounts — clean up resources
    }

    @Override
    protected ReactElement render() {
        return Html.div(
            Html.h3("Stopwatch: " + elapsed.getInt() + "s"),
            Html.button(running.getBool() ? "Stop" : "Start")
                .onClick(e -> running.setBool(!running.getBool()))
                .build()
        );
    }
}
----

[WARNING]
====
`ReactView` subclasses are *re-instantiated on every render*. State hooks declared as field initializers are called fresh each render, which is how React hooks work. Do not store mutable state outside of hooks.
====

=== ViewFactory Interface

`ReactView.toComponent()` and `ReactView.view()` accept a `ViewFactory`:

[source,java]
----

    ReactView create();
}
----

Use method references: `CounterView::new`, `StopwatchView::new`.


== Kotlin DSL

The `teavm-react-kotlin` module provides an idiomatic Kotlin experience using delegated properties, lambda-with-receiver builders, and coroutine integration.

=== Function Components with `fc()`

Define components using `fc()`, which provides a `ComponentScope` receiver:

[source,kotlin]
----
val Counter = fc("Counter") {
    var count by state(0)

    div {
        h2 { +"Count: $count" }
        button {
            +"Increment"
            onClick { count++ }
        }
    }
}
----

Render at the app root:

[source,kotlin]
----
fun main(args: Array<String>) {
    val root = ReactDOM.createRoot(
        HTMLDocument.current().getElementById("root"))
    root.render(component(Counter))
}
----

=== Delegated State Properties

State is declared using Kotlin's `by` delegation. Assignments trigger React re-renders:

[source,kotlin]
----
var count by state(0)           // IntStateDelegate
var name by state("Alice")      // StringStateDelegate
var active by state(true)       // BooleanStateDelegate
var price by state(9.99)        // DoubleStateDelegate
var items by stateList("a", "b")    // StringListStateDelegate
----

Functional updates are available via the delegate's `update()` method:

[source,kotlin]
----
val countDelegate = state(0)
var count by countDelegate

// Later, functional update:
countDelegate.update(StateHandle.IntUpdater { prev -> prev + 1 })
----

=== Effects and Coroutines

The `effect()` function wraps React's `useEffect` and provides a `CoroutineScope`:

[source,kotlin]
----
// Run after every render
effect {
    println("rendered")
}

// Run once on mount (pass no dependencies explicitly)
effectOnce {
    println("mounted")
    onCleanup { println("unmounting") }
}

// Run when dependencies change
effect(userId) {
    // re-runs when userId changes
    onCleanup { /* cleanup previous */ }
}

// Launch coroutines inside effects
effect {
    val job = launch {
        while (isActive) {
            delay(1000)
            ticks++
        }
    }
    onCleanup { job.cancel() }
}
----

==== `launchedEffect` (Compose-inspired)

Automatically launches and cancels a coroutine:

[source,kotlin]
----
launchedEffect {
    val data = api.fetchItems()  // suspend call
    items = data
}

launchedEffect(userId) {
    profile = api.fetchUser(userId)  // re-launches when userId changes
}
----

=== Flow Integration

Collect a Kotlin `Flow` into React state:

[source,kotlin]
----
val seconds by flow {
    var t = 0
    while (true) {
        emit(t++)
        delay(1000)
    }
}.collectAsState(initial = 0)
----

`collectAsState` is available for `Flow<Int>`, `Flow<String>`, `Flow<Boolean>`, and `Flow<Double>`. The flow starts collecting on mount and is automatically cancelled on unmount.

==== `produceState` (Compose-inspired)

[source,kotlin]
----
val profile by produceState("loading", userId) {
    value = api.fetchUser(userId).name
}
----

=== Typed Context

Type-safe React context without manual JSObject casting:

[source,kotlin]
----
// Create typed contexts
val ThemeContext = createStringContext("light")
val CountContext = createIntContext(0)
val AuthContext = createBoolContext(false)

// Provide a value
ThemeContext.provide("dark") {
    +App
}

// Consume in a component
val App = fc("App") {
    val theme: String = useContext(ThemeContext)
    div {
        +"Current theme: $theme"
    }
}
----

Factory functions: `createStringContext()`, `createIntContext()`, `createBoolContext()`.

=== Props Access

Read props passed to a component:

[source,kotlin]
----
val Greeting = fc("Greeting") {
    val name = propString("name")
    val age = propInt("age")

== Context API

React Context allows passing data through the component tree without prop drilling.

=== Java API

[source,java]
----
// Create a context with a typed default value
static final ReactContext THEME_CTX = ReactContext.create("light");

// Provide a value to descendants
THEME_CTX.provide("dark",
    childElement1,
    childElement2
);

// Consume in a component
static ReactElement themeConsumer(JSObject props) {
    String theme = THEME_CTX.useString();
    return Html.p("Current theme: " + theme);
}
----

==== `ReactContext` API

* `ReactContext.create()` -- create with no default value
* `ReactContext.create(String defaultValue)` -- create with String default
* `ReactContext.create(int defaultValue)` -- create with int default
* `ReactContext.create(boolean defaultValue)` -- create with boolean default
* `.useString()` / `.useInt()` / `.useBool()` -- consume the context value (typed)
* `.provide(String value, ReactElement... children)` -- wrap children with String provider
* `.provide(int value, ReactElement... children)` -- wrap children with int provider
* `.provide(boolean value, ReactElement... children)` -- wrap children with boolean provider

=== Kotlin Typed Context

See <<typed-context>> in the Kotlin DSL section. The Kotlin module provides type-safe wrappers:

[source,kotlin]
----
val ThemeCtx = createStringContext("light")

// Provide
ThemeCtx.provide("dark") {
    +App
}

// Consume

----


== React.memo

`React.memo()` wraps a component for shallow-prop comparison, skipping re-renders when props haven't changed:

[source,java]

=== Live-Reload Development Server

For iterative development, the project includes a live-reload dev server that watches source files, recompiles automatically, and refreshes the browser on every save.

[source,bash]
----
./dev.sh           # Build and serve on http://localhost:8080
./dev.sh 3000      # Custom port
----

Edit any `.java`, `.kt`, `.html`, or `.css` file in your preferred editor. The browser updates automatically.

==== How it works

The dev server (`DevServer.java`) is a single-file Java 21 program using only JDK built-in APIs:

1. **HTTP server** (`com.sun.net.httpserver.HttpServer`) serves `target/webapp/`
2. **File watcher** (`java.nio.file.WatchService`) recursively monitors all `src/` directories
3. **SSE endpoint** (`/__dev/events`) pushes reload notifications to the browser
4. **Script injection** — a live-reload `<script>` is injected into HTML responses automatically; no changes to `index.html` are required

When a source file changes, the server debounces for 400ms (to batch rapid saves), recompiles via Maven, and sends a `reload` event to all connected browsers. During compilation, the browser shows a "Recompiling..." overlay.

==== Build optimizations

The dev server activates the `dev` Maven profile, which tunes TeaVM for fast iteration:

[cols="1,2",options="header"]
|===
| Optimization | Effect

| Incremental TeaVM (`<incremental>true</incremental>`)
| Caches analysis results in `target/teavm-cache/`; only re-analyzes classes whose bytecode changed. This is the single biggest speedup.

| No source maps (`<sourceMapsGenerated>false</sourceMapsGenerated>`)
| Skips `.js.map` generation — saves I/O.

| No debug info (`<debugInformationGenerated>false</debugInformationGenerated>`)
| Skips debug metadata generation.

| Smart module detection
| If only `teavm-react-demo/src` changed, core and kotlin modules are not rebuilt.

| Static fast-path
| HTML/CSS-only changes in `src/main/webapp/` are copied directly to `target/webapp/` — Maven is not invoked at all.

| Maven Daemon (mvnd)
| Automatically detected and used if available. Eliminates JVM cold-start overhead (~2-3s per build).

| Parallel module builds (`-T 1C`)
| Core and kotlin modules build concurrently when both need rebuilding.
|===

== Troubleshooting

=== "React is not defined"

React must be loaded via `<script>` tags *before* `classes.js`. Ensure your HTML loads the React and ReactDOM CDN scripts first.

=== "classes.js not produced"

* Verify the `mainClass` in the TeaVM plugin configuration matches your entry point
* Ensure all dependencies are installed (`mvn install` on parent and core modules first)
* Check for compilation errors in the Maven output

=== "No ES6 spread in @JSBody"

TeaVM's `@JSBody` annotation does not support ES6 syntax including the spread operator (`...`). Use `.apply/.concat` instead:

[source,javascript]
----
// Instead of: React.createElement(type, props, ...children)
React.createElement.apply(null, [type, props].concat(children))
----

=== "Event handler not firing"

Event handlers must be set using the dedicated `React.setOn*` methods, not `React.setProperty`. If you're building custom components, use `ElementBuilder` or `DomBuilder` which handle this correctly. See <<critical-implementation-details>>.

=== "Rules of Hooks violation"

All hooks (`useState`, `useEffect`, `useRef`, etc.) must be called:

* At the *top level* of the render function (not inside conditionals or loops)
* In the *same order* every render
* Only inside a component render function (inside a `RenderFunction` or `ReactView.render()`)

This applies equally to Java and Kotlin. In `ReactView`, hooks declared as field initializers satisfy this requirement naturally.

=== "State not updating / stale closure"

Use functional updates when the new state depends on the previous value:

[source,java]
----
// WRONG — may use stale value in event handler
count.setInt(count.getInt() + 1);

// CORRECT — always gets the latest value
count.updateInt(n -> n + 1);
----

=== TeaVM Java API limitations

TeaVM provides a subset of the Java standard library. Notable unsupported features include:

* Reflection (beyond basic `Class` metadata)
* `java.nio` channels
* `java.net` networking
* Some `java.util.concurrent` classes

Check the https://teavm.org/docs/runtime/[TeaVM runtime documentation] for the full list of supported APIs.


== Contributing

=== Adding a New HTML Element

To add support for a new HTML element (e.g., `<dialog>`):

1. *`Html.java`* -- add `dialog(String text)` and `dialog(ReactElement... children)` methods
2. *`DomBuilder.java`* -- add a `public static final class Dialog extends DomBuilder` inner class

2. Create the handler interface with `@JSFunctor`
3. Add a `React.setOn*` method in `React.java` using `@JSBody`
4. Wire it into `ElementBuilder.java`, `DomBuilder.java`, and `HtmlBuilder.kt`
